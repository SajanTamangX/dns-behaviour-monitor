## [1.0.0] — 2026-05-05

First submission-ready release.

### Added
- End-to-end pipeline: Pi-hole (Docker, DNS on host port 5354) → traffic
  generation → log export → normalised parser → metrics → heuristics →
  Streamlit dashboard.
- Normalised parsed schema: `timestamp, domain, query_type, client,
  response_code, source`.
- Response-code recovery by correlating each `query[…]` line with the next
  `reply` / `cached` / `config` / `Pi-hole hostname` line for the same domain
  (NOERROR / NXDOMAIN / NODATA / SERVFAIL / REFUSED).
- Baseline metrics per dataset: query volume per 10-second bucket, top
  domains, TLD distribution, domain length distribution, label-count
  distribution, query-type distribution, response-code distribution.
- Four behavioural heuristics, all transparent and baseline-driven:
  - `long_domain` — length above max(baseline-max, mean + 2 σ); hard cap 60.
  - `burst_window` — bucket count above baseline p95 × 2.
  - `nxdomain_excess` — NXDOMAIN ratio ≥ 20 % AND ≥ 20 pp above baseline.
  - `rare_tld` — TLD seen in < 1 % of queries.
- Four built-in traffic profiles (`baseline`, `burst`, `nxdomain`,
  `longdomain`) driven by `scripts/generate_dns.py`.
- Streamlit dashboard with 13 modules: KPIs, rule status, signal/intel,
  spectrum, signature, frequency, depth, record-types, response codes, rule
  matrix, multi-dataset comparison, and detailed alerts.
- One-command demo (`scripts\demo.ps1`) that boots Pi-hole, generates four
  datasets, runs analysis, and opens the dashboard.
- Reproducible evaluation runner (`scripts\run_evaluation.ps1`) producing
  `outputs/evaluation_summary.md` and `outputs/evaluation_tables/*.csv`.
- Parser tests covering required columns, timestamp parsing, ≥ 95 % success
  rate, and response-code correlation.
- MIT licence, `.gitignore`, frozen `requirements.txt`.

### Notes
- Wording in the dashboard, code, and reports is strictly **behavioural /
  analytical**. The project does not make threat-detection or security
  claims.
- Pi-hole runtime state (`pihole/etc-pihole/`, `pihole/logs/`, FTL DB,
  `cli_pw`) is generated by the container at run-time and is not tracked
  in git.
